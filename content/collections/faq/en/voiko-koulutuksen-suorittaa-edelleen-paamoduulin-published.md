title: 'Can a training still be completed via the main module''s "published" link, even if the main module is no longer published?'
answer: '<p>Case 1: The main module is not published, but the submodules are published.</p><ul><li>If the user is logged in to the Builder client environment, they can complete the main module and submodules. In this case, statistics are collected from the submodules but not from the main module.</li><li>If the user is not logged in to the Builder client environment, they cannot complete the training and will receive an error message that the training is not published.</li></ul><p>Case 2: Neither the main module nor the submodules are published.</p><ul><li>If the user is logged in to the Builder client environment, they can only complete the main module. In this case, even if they complete the entire main module, no statistics are collected.</li><li>If the user is not logged in to the Builder client environment, they will receive an error message that the training is not published.</li></ul><p>In brief:</p><ul><li>Those logged in to the Builder client environment can complete modules even if they are not published.</li><li>Users not logged in to the Builder client environment cannot complete modules that are not published.</li><li>If the user can complete modules (e.g. logged in to the Builder client environment), statistics are only collected from the main and submodules that are published.</li></ul>'
