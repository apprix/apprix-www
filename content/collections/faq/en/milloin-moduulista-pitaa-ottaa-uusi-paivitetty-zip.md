title: 'When does a new, updated ZIP file need to be taken from a module and uploaded again to the LMS (learning management system)?'
answer: '<p>A new ZIP file must be uploaded again to the learning environment (LMS, Learning Management System) if someone at Apprix makes the following changes to the module content:</p><ul><li>a new image is added or replaced in an element that was not in your organisation''s image library at the time the ZIP file was downloaded</li><li>changes are made to the appearance of the training (e.g. font size is changed),</li><li>element functions are changed (recommended to confirm this with Apprix).</li></ul><p>If someone in your organisation makes changes to the content of the module or elements:</p><ul><li>a new image is added or replaced in an element that was not in your organisation''s image library at the time the ZIP file was downloaded</li><li>a completely new element is added to the module that was not yet in your organisation''s element selection when the ZIP file was downloaded from Builder,</li><li>when adding videos and audio files to the module.</li></ul><p>When is a new ZIP file not needed?</p><ul><li>when editing text content of elements,</li><li>when changing the order of elements,</li><li>after removing an element,</li><li>after adding an old element.</li></ul><p>About images in ZIP files</p><p>If you upload an image to the module and want it to be available to other users later and visible in the image library, tick the Show uploaded file in library checkbox. Otherwise the image will not be available to others through the image library in all modules in your client environment. Images added to the image library before downloading the ZIP file are automatically updated in the ZIP file if they are swapped into elements. If an image is not in the image library before downloading the ZIP file and it is later added to an element in the module, this requires uploading a new ZIP file to the learning environment.</p><p>Note! If the module''s ZIP file is used in a SCORM environment, it is possible to change images through Builder without needing to download a new ZIP file from the module. Just like with other public internet images, images in Builder''s image library can be linked directly by adding the image link to the Link to an image online field.</p><p>https://builder.apprix.fi/b/e/module_type_id/img/uploaded/IMAGENAME</p><p>Contact Apprix Helpdesk at helpdesk@apprix.fi to get the module type identifier (module type ID).</p> <img src="/files/docs/2020/07/changeimage.png" alt="" width="380" style="max-width:100%;height:auto;display:block;margin:0.75rem 0;">'
