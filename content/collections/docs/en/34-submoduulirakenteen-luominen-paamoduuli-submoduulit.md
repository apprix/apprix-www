<h3>When is it recommended to use a submodule structure?</h3><p>If you want your module to have a table of contents and the ability to divide content into sections or topic areas, use the Submodules functionality. In this case, the module structure is divided into a so-called main module and the submodules subordinate to it.</p><p>Benefits of a submodule structure:</p><ul><li><p>In long module structures, it becomes easier for the learner to grasp the overall picture.</p></li><li><p>Editing the module structure will be easier for you in the future, as the content is divided into sections rather than being one long continuous module.</p></li><li><p>You can define whether the module&#039;s sections are mandatory or optional for learners in terms of receiving a completion record.</p></li><li><p>You can conveniently add or detach sections in the future.</p></li><li><p><p></p><p></p><p>The course learner can complete the module in parts if the main module contains a Login element or if a user authentication method (e.g. SSO) is in use.</p><ul><li><p>The Login element is added to the main module before the <a href="/en/help/731-submenu-buttons"><strong>Submenu Buttons</strong></a> element, which serves as the menu page for submodules.</p></li><li><p>This means the learner can continue from the last fully completed section, and for example a long training does not need to be completed in one sitting.</p></li></ul></p></li></ul><h3>Main module containing two submodules in edit mode.</h3><h3>Submenu buttons element in Published state.</h3><h3>Creating a submodule structure</h3><p>Submodule structures can be created in many different orders, but in this example we start with the main module and then proceed to creating the submodules.</p><p><strong>Step 1: Creating the main module</strong></p><ol start="1"><li><p>First create a regular module as described in the Creating a Module (&quot;Create&quot;) section. Name this module in a way that describes its purpose and the course topic, as instructed in the <a href="/en/help/32-moduulin-nimeaminen"><strong>Naming a Module</strong></a> section. To make the structure easier to understand and work with later, it is recommended to add the word &quot;main module&quot; after the module name. For example: Safety Training – main module.</p></li><li><p>Add a <strong>Submenu Buttons</strong> element to the module structure, name it if needed, move it to the correct position in the module, save the changes by pressing <strong>Save</strong>, and open the element.</p></li><li><p><p></p><p></p><p>In the title field on the left side of the element you can name the menu (e.g. &quot;Menu&quot; or &quot;Training Menu&quot;), and in the text field below you can provide the learner with instructions for completing the structure. From the orange gear icon at the top of the element you can:</p><ul><li><p><p></p><p></p><p>add buttons to the menu for the submodules to be linked using the <strong>Add button</strong> button.</p><ul><li><p>Add a separate button for each submodule and name the buttons with the names of the upcoming submodules, so that linking the similarly named submodule to the correct button is straightforward.</p></li><li><p>It is recommended to name submodules descriptively based on their content or topic area.</p></li><li><p>Add an estimated time to complete the submodule after the name.</p></li></ul></p></li><li><p>define whether the submodules must be completed in the specified order (first first, etc.) or whether the learner can choose the order in which to complete the submodules.</p></li><li><p>define whether all submodules are mandatory or optional, so that the learner does not need to complete all submodules to proceed to the end and earn a completion record.</p></li></ul></p></li><li><p>When you are satisfied with the <strong>Submenu buttons</strong> element settings, close the element and exit the module.</p></li></ol><p><strong>Step 2: Creating a submodule</strong></p><ol start="1"><li><p><p></p><p></p><p>Create a new regular module using the <strong>Create</strong> button. Name the module with the name corresponding to its submodule button, so that linking the submodule button and its corresponding submodule together is straightforward.</p><ul><li><p>It is recommended to also add the name of the submodule structure/entire training after the submodule name (e.g. &quot;Introduction to Safety – Safety Training&quot;). This way, if the submodule is detached from the submodule structure for any reason, it is easy to find and link back to the correct main module.</p></li></ul></p></li><li><p><p></p><p></p><p>Now we begin converting the regular module into a submodule. Hover over the <strong>Submodule Settings</strong> button next to the <strong>Save</strong> button and click <strong>Change to submodule</strong> from the menu that opens. A window should now appear showing all the main modules to which submodules can be linked. Click the button name corresponding to your submodule from the menu you created, and the submodule will be linked to the main module.</p><ul><li><p>Note! If you try to convert a previously published module (<strong>Published</strong>) into a submodule, the <strong>Submodule Settings</strong> button will not be visible. Unpublish the module (<strong>Unpublish</strong>) if you want to convert it into a submodule.</p></li></ul></p></li><li><p><p></p><p></p><p>Repeat steps 1–3 for each submodule until the module structure is complete.</p><ul><li><p>Note! Only unpublished (<strong>Unpublished</strong>) modules can be linked to main modules.</p></li><li><p>For the submodule menu buttons to lead to the submodules when previewing or completing the submodule structure, the submodules must be published (<strong>Publish</strong>). A green dot will appear next to published submodules in the submodule list.</p></li></ul></p></li></ol><p><strong>Step 3: Editing submodules</strong></p><ol start="1"><li><p>You can edit submodules in the same way as a regular module. In the module edit view, open a submodule by clicking its name in the submodule list of the submodule structure.</p></li><li><p><p></p><p></p><p>You can return to the main module either by clicking the main module&#039;s title while a submodule is open, or by clicking the <strong>To main module</strong> button at the bottom left of the view.</p><p></p></p></li></ol><h3><strong>Main module containing two submodules in edit mode.</strong></h3><p><img src="statamic://asset::files::docs/2020/04/Esimerkki_submoduulit.png"></p><h3><strong>Submenu buttons element in Published state.</strong></h3><p><img src="statamic://asset::files::docs/2020/09/Esimerkkikoulutus_menu.jpg"></p><h3><strong>How is the Submenu Buttons element added to the main module and how are its submenu settings (Submenu settings) edited?</strong></h3><p><img src="statamic://asset::files::docs/2024/07/Main-module-submodule-1.gif"></p><h3><strong>How do you make a module into a submodule and link it to the main module?</strong></h3><p><img src="statamic://asset::files::docs/2024/07/Main-module-submodule-2.gif"></p><h3><strong>How are submodules linked to the main module published?</strong></h3><p><img src="statamic://asset::files::docs/2024/07/Main-module-submodule-3-1.gif"></p><h3><strong>A green dot appears next to published submodules in the submodule list.</strong></h3><p><img src="statamic://asset::files::docs/2024/07/published-submodules-green-balls.png"></p>
