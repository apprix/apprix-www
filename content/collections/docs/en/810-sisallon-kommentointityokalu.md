title: '8.10 Content Commenting Tool'

<p>Modules created with Apprix Builder can be commented on directly in the module's <strong>edit</strong> or <strong>preview mode</strong>. The commenting tool speeds up the module review and publishing process and makes it easier both to give and receive comments.</p><p>The commenting tool streamlines collaboration and allows comments from multiple different stakeholders to appear in the same place, at exactly the right point in the module. As a facilitator, you can also see comments in edit mode, making editing easier and faster.</p><h3>Commenting Tool Roles</h3><ul>
 <li><strong>Apprix admin</strong>: The Apprix responsible person with a super admin user role.</li>
 <li><strong>Facilitator:</strong> A customer's Builder user. Different permissions (view and edit) can be assigned to a facilitator through Builder, depending on what they need to be able to do.</li>
 <li><strong>Anonymous user:</strong> This user enters preview mode via a password-protected link provided to them, without authentication (view permissions).</li>
</ul><h3>How do I change a facilitator's permissions?</h3><p>If you have Admin rights, you can change a facilitator's module permissions in the <strong>Administrate</strong> menu if needed. Click the user's name in the list whose permissions you want to edit. Then click the "<strong>User's Module Edit &amp; Stats Permissions</strong>" <strong>Settings</strong> button. You can now update the facilitator's module permissions (None, View, Edit).</p><h3>Commenting Tool Preview Permissions</h3><p>Preview permissions can be restricted using a password and an expiry date.</p><h3>Facilitator Permissions</h3><h3>Edit permissions:</h3><ul>
 <li>Read access to all comments</li>
 <li>Permission to comment</li>
 <li>Comment management permissions (resolve, delete, restore)</li>
</ul><p><strong>View permissions, including anonymous users</strong>:</p><ul>
 <li>Read/comment permission, except for deleted comments.</li>
</ul><h3>Filters</h3><ul>
 <li>The commenting tool heading changes in the <strong>SHOW</strong> popup according to the options you select (All comments, Unresolved and deleted comments, Unresolved and resolved comments, Unresolved comments)</li>
 <li>If you are an anonymous user or a facilitator with only view permissions for this module, you cannot view the module in edit mode. In this case, you can view open and resolved comments in preview mode, but not deleted comments.</li>
 <li>Filters only change the user's view, not all users' views.</li>
 <li>Filters are not saved to memory.</li>
</ul><h3>Edit Element View</h3><p>Only facilitators with module edit permissions can open module elements and edit comments in <strong>Edit mode</strong>.</p><p>When an element is in edit mode, a green speech bubble appears in the upper right corner of the element. If there is a plus sign on the speech bubble, the element has no comments or unresolved comments.</p><p>If there are open comments, the speech bubble shows a number indicating the number of open comments.</p><p>If you click the speech bubble, the commenting tool appears to the right of the element with the heading <strong>Show: Unresolved comments</strong>.</p><p>Click the dropdown arrow button at the end of the commenting tool heading, and a smaller <strong>SHOW</strong> popup will appear. In addition to unresolved comments, you can choose what type of comments you want to view: <strong>Resolved</strong> and/or <strong>Deleted</strong></p><p><strong>Note!</strong> You can only see the <strong>Deleted</strong> button if you are a facilitator with module edit permissions.</p><h3>All Modules View</h3><p>A facilitator with module view permissions can preview a module they want to comment on directly from Builder's "<strong>All Modules</strong>" view by clicking the module's <strong>Preview</strong> button.</p><p>The module opens in preview mode, and for each element there is a green speech bubble icon in the upper right corner of the window. The element has no unresolved comments if the speech bubble icon has a plus sign. If the element has unresolved comments, the speech bubble shows a number.</p><p>Note that a facilitator with only view permissions can view resolved and unresolved comments, but not deleted ones. View permissions do not allow changing the status of a comment.</p><p>When a facilitator wants to reply to a comment, they need to click the reply arrow button below the comment and add their reply. When they want to create a new comment, they need to add the comment from the bottom of the comment window.</p><p>When a facilitator adds a new comment using the <strong>Add comment</strong> button, the new comment appears at the top of the comment window. If the facilitator clicks the reply arrow button and adds their reply to the <strong>Reply to comment</strong> text field, the reply appears below the reply arrow of the comment reply.</p><p>The number of unresolved comments is visible in the speech bubble on the right side of the comment popup.</p><h3>Anonymous Users</h3><ul>
 <li>Enter preview mode via a password-protected link provided to them, without authentication</li>
 <li>Enter a freely chosen username for their comment</li>
 <li>Have the same permissions as facilitators with preview permissions</li>
 <li>Cannot submit comments if the parameter "<strong>Enable preview sharing for this main module and its submodule</strong>" has not been activated in the "<strong>Share module preview</strong>" window, or if the link's date has expired.</li>
</ul><p><strong>Note!</strong> The username is saved in the tab's memory so it does not need to be entered again.</p><h3>How is the preview link for a module containing submodules presented to an anonymous user for commenting?</h3><ul>
 <li>In the module's <strong>Edit Module</strong> view, click the <strong>Preview</strong> button in the upper right corner of the window.</li>
 <li>Select the <strong>Share</strong> option, which opens the "<strong>Share module preview</strong>" popup.</li>
 <li>Activate the "<strong>Enable preview sharing for this main module and its submodules</strong>" button.</li>
 <li>Set a password for sharing the preview.</li>
 <li>Set the expiry date for sharing the preview (at most one month from today).</li>
 <li>Click the <strong>Copy link and message</strong> button.</li>
 <li>Deliver the password and link you created to the anonymous user.</li>
</ul><p>We recommend that the anonymous user opens a tab in incognito mode and enters the link and password. The anonymous user can now add and read comments, navigate from the main module to a submodule and back to the main module. Note that if the link has expired, the module cannot be viewed.</p><p>It is important to note that an anonymous user can add and read comments, but cannot delete or resolve them. This restriction is in place to maintain the integrity of the feedback process.</p><h3>How is a comment added to an element?</h3><p>Add a comment at the bottom of the commenting tool under the <strong>Add comment</strong> section and click <strong>Send</strong>. You will notice that your initials, name, date, and comment text appear in the text box at the top of the commenting tool.</p><p><strong>Note!</strong> Comments are shown from newest to oldest. Comment replies are shown in the order they were made.</p><p>If you are a facilitator with module edit permissions, you will see an icon with three dots to the right of the text box. Click the icon, and a small popup will open. The window contains the options <strong>Resolve</strong> and <strong>Delete</strong>. You can change the status of a comment by clicking either of these options.</p><p>If a comment has been resolved or deleted, you can undo the resolution and/or deletion by clicking this three-dots icon again and selecting the desired option (<strong>Undelete</strong>/<strong>Unresolve</strong>).</p><p>You will now notice that a reply arrow icon appears below an unresolved comment whenever a comment or reply is being created. Click the reply arrow icon, and a <strong>Reply to comment</strong> text box will appear below the reply arrow icon. You can write your reply and choose <strong>Send</strong> if you want to reply to the comment. Otherwise, select <strong>Cancel</strong>.</p><p>If you click the <strong>Send</strong> button, your initials, name, date, and comment will appear below the reply arrow icon. Note that a new reply arrow icon appears below your comment.</p><h3>How can a facilitator with module edit permissions resolve a comment?</h3><p>Suppose there is one or more replies to a comment. If a facilitator with module edit permissions clicks the comment's initials or the <strong>Resolve</strong> button, they resolve the comment along with all its replies. When a comment is resolved, an <strong>OK</strong> icon appears over the icons that previously contained the initials.</p><p>If the facilitator instead clicks only a reply, they resolve or delete only the reply, not the comment.</p><h3>Note!</h3><ul>
 <li>Only facilitators with module edit permissions can resolve, unresolve, delete, and restore comments.</li>
 <li>The green speech bubble icon shows if you have one or more unresolved comments.</li>
 <li>A comment cannot be replied to if it has been resolved or deleted.</li>
 <li>An empty comment cannot be submitted</li>
</ul><ul>
 <li>The status of a deleted comment's reply cannot be changed until the comment has been restored.</li>
</ul><h3>Edit Module View</h3><p>When you close an element with unresolved comments in edit mode, the number of unresolved comments is displayed at the top of the element in the <strong>Edit Module</strong> view. In addition, a <strong>Comments</strong> speech bubble appears on the right side of the window, showing the total number of unresolved comments (comments from the main module and submodules all together).</p><p>Click the <strong>Comments</strong> speech bubble on the right side of the window to open the <strong>Comments</strong> popup. The window lists the names of the modules (main module and submodules) that contain unresolved comments. When all comments have been resolved, the <strong>Comments</strong> speech bubble disappears.</p><p>In the <strong>Browse Submodules</strong> menu, the number of unresolved comments is shown next to the element, and the total number of unresolved comments is shown in the top corner. The speech bubble disappears if there are no comments.</p><h3>All Modules View</h3><p>A speech bubble icon next to a main module means that the main module (or its draft) or at least one submodule (or its draft) has unresolved comments.</p><p>A speech bubble icon next to a submodule means that the submodule (or its draft) has an unresolved comment.</p><h3>Drafting Feature</h3><ul>
 <li>In a draft, comments are transferred from the published module to the new draft.</li>
 <li>If a draft version of the module exists, the commenting tool is not available with the published version, because comments have been moved to the draft.</li>
 <li>When you delete a draft or an element that contains comments, the comments are permanently deleted.</li>
 <li>The Browse submodules menu always shows the published submodule, not the draft.</li>
</ul><h3>General Notes</h3><ul>
 <li>Commenting does not leave entries in the audit trail.</li>
 <li>The Builder view and commenting tool do not update in real time if another facilitator is, for example, writing comments at the same time.</li>
</ul><h3>Click the video to enlarge it</h3><h3>All Modules and Edit Element Views</strong>
