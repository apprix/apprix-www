<h4><strong>General</strong></h4>
Modules created with <strong>Apprix Builder</strong> can be commented directly in the <strong>edit</strong> or <strong>preview</strong> <strong>mode</strong> of the module. The <strong>commenting</strong> <strong>tool</strong> speeds up the process of reviewing and publishing modules and makes it easier to both give and receive comments.

The commenting tool streamlines collaboration and allows comments from multiple stakeholders to appear in the right place in the module. As a facilitator, you can also see the comments in edit mode, making editing more accessible and faster.
<h4><strong>Roles in the commenting tool</strong></h4>
<ul>
 <li><strong>Apprix admin</strong>: the person in charge of Apprix with the super admin user role.</li>
 <li><strong>Facilitator</strong>: user of the customer. Through Builder, the facilitator can be assigned different permissions (view and edit) depending on what you want to enable them to do.</li>
 <li><strong>Anonymous user</strong>: this user enters the preview mode via a password-protected link without authentication (only viewing rights).</li>
</ul>
<strong> </strong>
<h4><strong>How do I change the facilitator permissions?</strong></h4>
If you have Admin rights, you can change the access rights to the facilitator modules in the <strong>Administrate</strong> menu if necessary. In the name list, click on the name of the facilitator whose access rights you want to modify. Then click the "<strong>User's Module Edit &amp; Stats Permissions</strong>" -<strong>Settings</strong> button. You can now update the facilitator module's permissions (None, View, Edit).

<strong>Preview rights for the commenting tool</strong>

Preview rights can be limited by password and expiration date.

<strong> </strong>
<h4><strong>Facilitator access rights</strong></h4>
<strong> Editing rights:</strong>
<ul>
 <li>Reading access to all comments</li>
 <li>Commenting rights</li>
 <li>Comment management rights (resolve, delete, return)</li>
</ul>
Viewing rights, <strong>including anonymous users</strong>:
<ul>
 <li>Read/comment rights, except for deleted comments.</li>
</ul>
<h4><strong>Filters</strong></h4>
<ul>
 <li>The title of the comment tool changes in the <strong>SHOW</strong> pop-up window according to the options you select (All comments, Unresolved and deleted comments, Unresolved and resolved comments, Unresolved comments)</li>
 <li>If you are an anonymous user or a facilitator with only viewing rights to this module, you cannot view the module in edit mode. In this case, you can view open and resolved comments in preview mode but not deleted comments.</li>
 <li>Filters only change the view of the user, not the view of all users.</li>
 <li>Filters are not saved.</li>
</ul>
<h4><strong>Edit Element view </strong></h4>
Only <strong>facilitators</strong> with <strong>edit access</strong> to the module can open the module's elements and <strong>edit comments</strong> in <strong>Edit mode</strong>.

When an element is in edit mode, a green speech bubble appears in the upper right corner of the element. If there is a plus sign above the speech bubble, there are no comments or unresolved comments in that element.

If there are open comments, the speech bubble will display a number indicating the available comments.

If you click on the speech bubble, the comment tool will appear on the right side of the element with the title <strong>Show: unresolved comments</strong>.

Click the drop-down arrow button at the end of the comment tool title; a smaller <strong>SHOW</strong> pop-up window will appear. In addition to the unresolved comments, you can choose which type of comments you want to view: Resolved and/or Deleted.

<strong>Note!</strong> You will only see the <strong>Deleted</strong> button if you are a facilitator with module editing rights.

<strong> </strong>
<h4><strong>All Modules -view</strong></h4>
A facilitator with module <strong>viewing rights</strong> can directly preview the module they wish to comment on from the "<strong>All Modules</strong>" view in Builder by clicking the module <strong>Preview</strong> button.

The module will open in preview mode, with a green speech bubble icon in each element's top right corner of the window. There are no unresolved comments on an element if the speech bubble icon has a plus sign. The speech bubble icon has a number if an element has unresolved comments.

Note that a facilitator with view-only access can only view resolved and unresolved comments, not deleted ones. Viewing rights do not allow you to change the status of a comment.

When the facilitator wants to reply to a comment, they must click the reply arrow below the comment and add a reply. When they want to create a new comment, they must add it at the bottom of the comment window.

When the facilitator adds a new comment using the <strong>Add comment</strong> button, the new comment will appear at the top of the comment window. If the facilitator clicks the reply arrow button and adds their replies in the <strong>Reply to Comment</strong> text box, they will appear below the reply arrow in the comment reply.

The number of unresolved comments can be seen in the speech bubble to the right of the comment pop-up window.

<strong> </strong>
<h4><strong> </strong><strong>Anonymous users</strong></h4>
<ul>
 <li>Enter the module's preview mode without authentication via the password-protected link provided to them.</li>
 <li>Enter a nickname of their choice for their comments</li>
 <li>Have the same rights as facilitators with preview rights</li>
 <li>Cannot post comments if the parameter "<strong><em>Enable preview sharing for this main module and its submodule </em></strong>in the<strong>" Share module preview</strong>" pop-up window is not activated or the date in the link has expired.</li>
</ul>
<strong>Note!</strong> The nickname will be saved in the tab memory so that it does not need to be re-typed.
<h4><strong>How do we deliver the preview link of a module that contains submodules to an anonymous user for commenting?</strong></h4>
<ul>
 <li>In the module <strong>Edit module view</strong>, click the <strong>Preview</strong> button at the top right of the window.</li>
 <li>Choose  <strong>Share.</strong> The “<strong>Share module preview</strong>” pop-up window is opened.</li>
 <li>Activate the “<strong>Enable preview sharing for this main module and its submodules</strong>” button.</li>
 <li>Set the password to share the preview.</li>
 <li>Set the end date for sharing the preview (up to one month from today)</li>
 <li>Click on the <strong>Copy link and message button</strong>.</li>
 <li>Deliver the password and the link to the anonymous user.</li>
</ul>
We suggest the anonymous user open the tab in incognito mode and enter the link and password.

The anonymous user can now add and read the comments, move from the main module to the submodule, and then back to the main module. Viewing the module will not be possible if the link validity date expires.

It's important to note that the anonymous user can add and read comments but cannot delete or resolve them. This limitation is in place to maintain the integrity of the feedback process.
<h4></h4>
<h4><strong>How to add a comment to an element?</strong></h4>
Add a comment to an element at the bottom of the comment tool under <strong>Add comment</strong> and click <strong>Send</strong>. You will notice that your initials, name, date and comment text will appear in the text box at the top of the comment tool.

<strong>Note!</strong> Comments will appear from newest to oldest. Responses to comments are displayed in the order in which they were replied.

If you are a <strong>facilitator with module editing rights</strong>, you will see an icon with three dots to the right of the text box. Click on the icon to open a small pop-up window. The window contains the options <strong>Resolve</strong> and <strong>Delete</strong>. You can change the status of the comment by clicking on either of these options.

Suppose the comment has been resolved or deleted. In that case, you can undelete and/or delete the comment by clicking this three-dot icon again and selecting the option you want (<strong>Undelete</strong>/<strong>Unresolve</strong>).

You will now notice that a reply arrow icon appears below an unresolved comment each time a comment or reply is created. Click on the reply arrow icon, and the "<strong>Reply to comment</strong>" text box will appear below the reply arrow icon. You can type your reply and select <strong>Send</strong> to reply to the comment. Otherwise, select <strong>Cancel</strong>.
<h4><strong>How can a facilitator with module editing rights resolve a comment?</strong></h4>
Assume that there are one or more responses to a comment. If the facilitator with module editing rights clicks on the comment's initials or the Resolve button, they will resolve the comment with all its answers. When the comment is resolved, an <strong>OK</strong> icon will appear over the icons that previously contained the initials.

If, on the other hand, the facilitator clicks only on the reply, they will resolve or delete only the reply and not the comment.

<strong>Note!</strong>
<ul>
 <li>Only facilitators with module editing rights can resolve, leave unresolved, delete and withdraw comments.</li>
 <li>A green speech bubble icon indicates if you have one or more unresolved comments.</li>
 <li>You cannot reply to a comment if it has been resolved or deleted</li>
 <li>You cannot send an empty comment</li>
 <li>The reply status of a deleted comment cannot be updated until the comment has been restored.</li>
</ul>
<h4><strong>Edit Module view</strong></h4>
When you close an element with unresolved comments in edit mode, the number of unresolved comments is displayed in the <strong>Edit Module</strong> view at the top of the element. In addition, a <strong>Comments</strong> speech bubble appears on the right side of the window, containing the total number of unresolved comments (comments from the main module and submodules combined).

Click on the <strong>Comments</strong> speech bubble on the right side of the window to open the <strong>Comments</strong> pop-up window. The window lists the names of the modules (main module and submodules) that contain unresolved comments. When all comments have been resolved, the <strong>Comments</strong> speech bubble will disappear.

The <strong>Browse Submodules</strong> menu displays the number of unresolved comments for the element and the total number of unresolved comments in the upper corner. The speech bubble is cleared if there are no open comments.

<strong> </strong>
<h4><strong>All Modules view</strong></h4>
The speech bubble icon for a main module means unresolved comments in the main module (or its draft) or one of the submodules (or its draft).

A speech bubble icon for a submodule means an unresolved comment in the submodule (or its draft).
<h4><strong>Draft feature</strong></h4>
<ul>
 <li>In a draft, comments are moved from a published module to a new draft.</li>
 <li>If there is a draft version of the module, the commenting tool is unavailable in the published version because the comments have been moved to the draft.</li>
 <li>When you delete a draft or an element with comments, the comments are permanently removed.</li>
 <li>The <strong>Browse submodules</strong> menu always shows the published submodule, not the draft.</li>
</ul>
<strong>General remarks</strong>
<ul>
 <li>The commenting tool does not leave an entry in the audit trail.</li>
 <li>Builder view and the commenting tool are not updated in real-time if, for example, another facilitator writes comments simultaneously.</li>
</ul>
