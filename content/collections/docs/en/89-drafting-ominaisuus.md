<strong>Drafting</strong> feature distinguishes the published module from a new draft version.
<ul>
 <li>It enables you to edit a draft version of a published module so that the original published module can still be used in its original format while you are making changes. The end users will not see any updates in the published version until you decide to publish the new draft version.</li>
 <li>If changes to the module are to be traced, the Audit Module feature should be activated in the module. This is done by selecting the <strong>Audit Module</strong> option from the module gear icon on the top of the window in the <strong>Edit Module</strong> view.</li>
 <li>Once the <strong>Audit Module</strong> is activated and the module is published, a snapshot of the current version is created.
<ul>
 <li>The snapshot can be viewed and compared with different publications or with a new draft version of the same module.</li>
 <li>This can be very useful if, for example, the authorities may require evidence that the training has followed given regulations during a certain period.</li>
 <li>With the Audit Module feature, traces of changes done in the module are left in a list that shows, for each change,
<ul>
 <li>what kind of change has been made,</li>
 <li>to which element,</li>
 <li>who has made the change</li>
 <li>and when the change was made.</li>
 <li>The list of changes can be exported to a file in PDF, CSV, and Excel formats.</li>
</ul>
</li>
</ul>
</li>
</ul>
<strong> </strong>
<h4><strong>Drafting Buttons</strong></h4>
If the Drafting feature is enabled, the following buttons will appear in the <strong>Edit Module</strong> view after publishing the module:
<ul>
 <li><strong>“Drafting”</strong> button at the top of the window. When you click on the <strong>Drafting</strong> button, a drop-down box opens. Here you can choose from two options: <strong>Publish draft</strong> and <strong>Discard draft</strong>.</li>
 <li>"<strong>Draft</strong>" pink button (contains draft-related instructions) on the right side of the window. You are now in front of the draft version of the module. End users can use the published version in its original form when you update its content. Note! End users will not notice the changes until you decide to release a new version of this module.</li>
 <li>"<strong>SAVE DRAFT</strong>" button at the bottom of the tab. Select this option whenever you want to save your changes to a new draft version.</li>
</ul>
<strong>Note</strong>!
<ul>
 <li>In the <strong>All Modules</strong> view, you will see the content of the module’s published version.</li>
 <li>In the <strong>Edit module</strong> view, you will see the <strong>DRAFT</strong> content of the module.</li>
</ul>
<h4></h4>
<h4><strong>Drafting</strong>: <strong>”Publish Draft” and ”Discard Draft” Buttons</strong></h4>
When you click on the <strong>Drafting</strong> button, a drop-down menu opens. Here you can choose from two options: <strong>Publish draft</strong> and <strong>Discard draft</strong>.
<ul>
 <li><strong>Publish draft</strong>: Select this button when you are ready to publish a new draft version of the module.</li>
 <li><strong>Discard draft</strong>: Select this button if you want to discard your changes to the current draft version. This action reverts the changes back to the previously released version.</li>
</ul>
<h4></h4>
<h4><strong>How to Use the Drafting and Audit Modules Features</strong></h4>
If you wish to create a viewable version history for published modules:
<ol>
 <li>Activate the <strong>Audit Module</strong> feature from the gear symbol in the module editing view.</li>
 <li>Activate the <strong>Publish</strong> button in the module editing view once the module is ready to be published.</li>
 <li>After publishing the module, you can view the first saved version of the module in the version history by clicking the <strong>Last modified</strong> text link in the <strong>All Modules</strong> view, which will open the <strong>About Module</strong> pop-up window.</li>
 <li>Scroll down the <strong>About Module</strong> window, which contains the version history of the module. At the bottom of the window, you can see the <strong>Module Audits change history</strong> (saved at the time of publication), and below it you can see the newly published version of the module. Click the <strong>Open</strong> button and the newly published version can be viewed in the preview mode.</li>
 <li>When the module has been published, a pink <strong>Draft</strong> info bar appears in the upper left corner of the module’s editing view. The clickable info bar contains general instructions for using the <strong>Drafting</strong> If necessary, at this stage you can start editing a new draft version of the module. The original published version can still be used while changes are made to the new draft version. The changes will not be visible to those executing the module until the new draft version is released.</li>
 <li>Please note that in the <strong>All Modules</strong> view you can see only the published version of the module. It is possible to view the draft version only in the editing view of the module.</li>
 <li>Whenever a module is <strong>republished</strong>, a new version of the module will be added to the <strong>About Module</strong> window if the Audit Module feature is enabled.</li>
 <li>Note! If you duplicate a module with the Audit module feature activated, it is automatically activated in the new copy of the module as well.</li>
</ol>

<h4><strong>Draft and Published Module Preview</strong></h4>
By using the Preview drop-down menu, you can also open and share a preview link of either a published module or a draft version.

Preview options:
<ul>
 <li><strong>Draft module preview</strong>: Share / Open</li>
 <li><strong>Published module preview</strong>: Share / Open</li>
</ul>
<strong>Note</strong>! The same instructions apply to both “Share” and “Open” options as described in <em>Chapter 6.1. Previewing the module</em>.
