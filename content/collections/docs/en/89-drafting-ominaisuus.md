<p>The Drafting feature separates the published module from a draft version of an already published module.</p><ul>
 <li>It enables working on a draft version of a published module without the changes automatically appearing in the published, already potentially active version. Changes do not appear in the module's published view until the draft is published.</li>
 <li>If you want the Drafting change history to be recorded with the Audit Module additional feature, it must be enabled from the <strong>gear icon</strong> at the top of the module edit view.</li>
 <li>When the Audit Module feature is activated and the module is published, a saved copy is created (version history). The original version and later edited and published versions can be compared with each other.
<ul>
 <li>This feature can be useful, for example, when authorities require evidence of compliance with legally mandated instructions or regulations in a specific training over a given period.</li>
 <li>With the Audit Module feature, changes are recorded as a list in the change history, showing what changes were made to the module or an individual element, who made the changes, and when. The change history can be downloaded in PDF, CSV, or Excel format.</li>
</ul>
</li>
</ul><h3>Drafting buttons</h3><p>If the Drafting feature is enabled, after publishing the module, the following buttons appear in the <strong>Edit Module</strong> tab:</p><ul>
 <li>A "<strong>Drafting</strong>" button at the top of the tab. When you click the Drafting button, a dropdown opens. Here you can choose between two options: <strong>Publish draft</strong> and <strong>Discard draft</strong>.</li>
 <li>A pink "<strong>DRAFT</strong>" button on the right side of the tab. You are now in the draft version of the module. End users can use the published version in its original form at the same time as you are updating the module content. By hovering the mouse over the question mark on the button, you can also open the instructions related to the draft. <strong>Note!</strong> End users will not notice changes until you publish the new version of this module.</li>
 <li>A "<strong>SAVE DRAFT</strong>" button at the bottom of the tab. Select this option whenever you want to save the changes you have made to the new draft version.</li>
</ul><p>Note that in the <strong>All Modules</strong> view of the module, you see the published version of the module, and in the <strong>Edit Module</strong> view you see the DRAFT version of the module.</p><h3>Drafting: Publish draft and Discard draft buttons</h3><p>When you click the <strong>Drafting</strong> button, a dropdown opens. Here you can choose between two options: Publish draft and Discard draft.</p><ul>
 <li><strong>Publish draft</strong>: Select this button when you are ready to publish the new draft version of the module.</li>
 <li><strong>Discard draft</strong>: Select this button if you want to discard the changes you have made to the current draft version. This action restores the changes back to the previously published version.</li>
</ul><h3>Using the Drafting and Audit Module features</h3><p>If you want to create a reviewable version history for published modules:</p><ol>
 <li>Activate the <strong>Audit Module</strong> feature from the <strong>gear icon</strong> in the module edit view.</li>
 <li>Activate the <strong>Publish</strong> button in the module edit view when the module is ready to be published.</li>
 <li>After publishing the module, you can view the first saved version of the module. You can access the version history by clicking the <strong>Last modified</strong> text link in the All Modules view, which opens the <strong>About Module</strong> popup.</li>
 <li>Scroll down the <strong>About Module</strong> window, which contains the module's version history. At the bottom of the window you can see the <strong>Module Audits</strong> change history (saved upon publishing). Below this you can see the version of the module that was just published. Click the <strong>Open</strong> button and the just-published version can be viewed in preview mode.</li>
 <li>When the module is published, a pink <strong>Draft</strong> info bar appears in the upper left corner of the module edit view. The clickable info bar contains general instructions for using the Drafting feature. If needed, you can at this point begin editing a new draft version of the module. The original published version can still be used while changes are being made to the new draft version. Changes will not be visible to module participants until the new draft version is published.</li>
 <li>Please note that in the All Modules view you can only see the published version of the module. Only in the module edit view is it possible to view the draft version.</li>
 <li>Every time the module is republished, a new version of the module is added to the About Module popup, if the Audit Module feature is activated.</li>
 <li>Note! If you duplicate a module that has the Audit module feature enabled, it is automatically enabled in the new copy of the module as well.</li>
</ol><h3>Handling module drafts</h3><p>Using the Preview dropdown, you can also open and share a preview link for either the published module or the draft version.</p><p>Preview options:</p><ul>

 <li><strong>Published module preview</strong>: <strong>Share</strong> / <strong>Open</strong></li>
