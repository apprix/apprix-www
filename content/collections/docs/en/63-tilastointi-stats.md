title: '6.3 Statistics (Stats Option)'
---
When a module has been published, it starts collecting data. You can view module statistics by clicking the <b>Stats </b>button on the left side of published modules in <b>All Modules </b>or the<strong> Edit Module</strong> view. Statistics gather all performance data in an up-to-date manner.

<b>Note!</b> Collecting user data requires that the module have a Login element or another authentication method (e.g., Email Registration, Consumer Login, SSO/Single sign-on, or Tax number check) in the module settings (see the "User Authentication" tool from the gear icon in the module editing view).

Note! The data are collected separately for each module and submodule. 
<h4>Functions in Stats Window:</h4>
<ul>
 <li>In the <strong>Overview</strong>, you may view a summary of module statistics.</li>
 <li><b>Duration</b> is the time that the trainee has spent in the module.</li>
 <li>You may view all trainees who have successfully passed the module from the <strong>Users</strong> tab.</li>
 <li>The <strong><b>Big Data</b> view shows the statistics of all end-users who have attended the course.</strong>
<ul>
 <li><b>Show Finished</b> button: shows only successfully finished attempts (Finished = 1)</li>
 <li><b>Show Unfinished</b> button: shows both finished and unfinished attempts (Finished = 0)</li>
 <li aria-level="1"><b>Show Archived button: if an element containing an exercise (e.g. True / False, CheckBox) has been removed from the module after publication, the data collected by this exercise(s) will be displayed in the statistics when this option is selected.</b></li>
 <li>In the <b>Big Data</b> view, click the Export button to filter and export the data you want to analyze.
<ul>
 <li>You may also use the <strong>Search</strong> field to filter data.</li>
</ul>
</li>
 <li><b>The progress column shows the trainee's progress in the module (e.g., 3 = the </b>third element in the module). 
<ul>
 <li>If the user moves backwards in the module and logs out, Builder will save the last "location" where s/he has been (the element) in the statistics.</li>
 <li>If the user has progressed to the end of the module (until the Finish element), it will be marked "Finished = 1." In this case, "Finished = 1" does not change even if the user moves backwards and logs out afterwards.</li>
</ul>
</li>
 <li><b>Score</b> / <b>Max. Score</b> column: <strong>user points/max</strong><strong>. possible points</strong> in the module. 
<ul>
 <li>The score is reset if the user returns to the module later on and starts the module again.</li>
</ul>
</li>
</ul>
</li>
 <li><strong>Feedback </strong>tab: If the <b>Feedback</b> element is added to the module, feedback from the trainees will be shown in the Statistics <b>Feedback</b> view.
<ul>
 <li><strong>Note!</strong> The free-form, written feedback shown in the <b>Feedback</b> view has been entered in the open text field at the right side of the <b>Feedback</b> element.</li>
 <li>Open the Big Data view if you wish to view all feedback in the Feedback element. </li>
</ul>
</li>
</ul>
<b>Tip!</b> If your organization needs even more detailed, customized reports, consider the <a href="#extendedreports"><b>Extended Reports feature.</b></a>

<h6 class="p1"><b>Click on the images to enlarge them.</b></h6>

<h6><em>If the module was successfully completed, the value in the Finished column is 1. If the module was not completed successfully, the value in the Finished column is 0.</em></h6>

<h6><em>In the Feedback view, you can see the feedback of the trainees that have successfully completed the training.</em></h6>
