<p>The purpose of the <strong>Summary Reports</strong> reporting view is to consolidate module completions in one place so that facilitators can monitor them.</p><p>Summary Reports access rights are granted in the <strong>Administrate</strong> → <strong>Users</strong> tool.</p><p><strong>Note</strong>! A facilitator can also grant access rights to external users.</p><p>An external user's facilitator can be set in the <strong>Administrate → Users</strong> tool with the settings <strong>Module Access</strong> = <strong>None</strong> and <strong>Summary Report</strong> = <strong>On</strong>, giving the external user access only to reporting.</p><p>When the <strong>Administrate</strong> → <strong>Users</strong> setting is activated for a facilitator, a new <strong>Reports</strong> button appears in their Builder main menu.</p><p>When the facilitator clicks the <strong>Reports</strong> button, the reporting view opens. Here they can select multiple filters by clicking the <strong>Filters</strong> button in the upper left corner of the window.</p><p>Filters are customer-specific. They can be, for example, <strong>Emails</strong> (the email address of the person who completed the course), <strong>Modules</strong> (only data from published modules is available in reports), <strong>Date Range</strong> (the date range for completions), or <strong>Finished</strong> (if selected, only completions that have finished the training are shown; if not selected, all completions are shown, including incomplete ones).</p><p>Once the facilitator has set the filters and obtained the desired report, they can download it in <strong>CSV</strong> or <strong>Excel</strong> format.</p><h3>Note!</h3><p>The more advanced version of the Summary Reports feature is the <strong>Extended Reports</strong> feature.</p><p>More information about this feature can be found at:</p><h3>8.8 Extended Reports</h3><p><a href="/en/help/88-extended-reports">8.8 Extended Reports</a></p><h3>Summary Reports access rights are granted in the <strong>Administrate</strong> → <strong>Users</strong> tool.</h3>
