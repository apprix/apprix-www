You can add modules to a module group so that course participants can complete any language modules in the training. This means that the <strong>Nagging</strong> (<strong>Participants &amp; Reminders</strong>) <strong>settings</strong> are shared with modules in the same module group, including the email addresses added to the Nagging and Nagging reminder settings. Other settings, such as User authentication, Expiration settings, and statistics, are still module-specific.

<strong>Module Group and Nagging editing instructions</strong>

Add a module to a module group as follows:
<ol>
 <li>Open the module to be added to the module group in <strong>Edit mode</strong></li>
 <li>Select "<strong>Module Groups...</strong>" from the top wheel menu.</li>
 <li>Create a new group for the module by selecting "<strong>Add to new group</strong>" or add the module to an existing group by selecting "<strong>Add to existing group.</strong>"</li>
</ol>
Edit the <strong>Nagging</strong> (aka <strong>Participants and reminders</strong>) settings as follows to add the necessary email addresses and enable Nagging auto-reminder:
<ol>
 <li>Open the module to be added to the module group in <strong>Edit mode</strong></li>
 <li>Select "<strong>Nagging</strong>..." from the wheel icon.</li>
 <li>Select "<strong>Add...</strong>" to add any missing email addresses of the course participants to be reminded.</li>
 <li>Enter the email addresses in the text field for the different lines and click "<strong>Add participants</strong>."</li>
 <li>Edit the Nagging automatic reminder settings by clicking "<strong>Automatic Reminder Settings...".</strong></li>
 <li>Turn on "<strong>Use automated email reminders</strong>" and edit the reminder interval in the drop-down menu "<strong>Send reminders until course completed every</strong>" and the maximum number of reminders in the drop-down menu "<strong>Max. number of reminders per user</strong>".</li>
 <li>Then, enter the email's subject in the "<strong>Subject</strong>" field and the content of the email in the large text field below. For more information on editing the content, including how to use variables, see the question mark icons at the back. For example, the module address replaces the <strong>{link} </strong>field. If the module is linked to MyCourses, we recommend replacing <strong>{link}</strong> with the MyCourses course link. Such a course link lets the user choose which language versions to take.</li>
</ol>

<strong>Hungry for more information? Below are more questions and answers.</strong>

<strong>What link can a Builder course participant use when there are several language versions of the training?</strong>

<strong> </strong>When there are multiple language versions, we recommend providing the trainee with a link to choose between the language versions before the actual training. The link should lead to the place where the language versions are linked—for example, a link to a Landing page or MyCourses course.

It is essential to use this link in reminder messages for features such as <strong>Nagging</strong> and <strong>Expiration</strong> so that the course participant can find the place to complete the training.

When the language versions of the training are linked to MyCourses, we recommend providing the course participant with a MyCourses course link. Where the language versions of the training are linked to the Landing Page module, we recommend that you give the course participant a direct link to the Landing Page module.

<strong>What is the difference between a MyCourses course, a Landing Page training button and a Builder Module Group?</strong>

The MyCourses course, Landing Page training button, and Module Group are ways of grouping different modules (e.g., language version modules) into a single entity. When one of the modules linked in this way is completed, the whole is considered completed.
<ul>
 <li>In MyCourses, when a module linked to a course is completed, the MyCourses course set is considered completed.</li>
 <li>When a module linked to a Landing Page training button is completed, the training button will appear as completed.</li>
 <li>When modules in a Module Group are completed, Builder features such as Nagging will consider the whole as completed, and the completion of the whole will no longer be remembered.</li>
</ul>
<strong>How do MyCourses course links work?</strong>

When a course participant opens a MyCourses course link, they will be asked to log in to MyCourses if they are not already logged in. They will then see a language selection dialogue for the MyCourses course, where they can choose the language they wish to complete the course.

<strong>How do I send a reminder message to MyCourses course participants who have not yet completed a MyCourses course with two or more language versions?</strong>

Builder modules linked to a MyCourses course should be grouped using the Module Groups property. The Nagging reminder settings should be configured for this group of modules so that the reminder message contains a link to the MyCourses course.
