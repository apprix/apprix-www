title: '7.30 Pass/Fail & Pass/Fail Always'

<p>With the <strong>Pass/Fail</strong> element, you can notify the learner that they have not passed the module's final test and explain the next steps. The element can be added either immediately after the final test or at the end of the module before the final summary (Commitment, Feedback), if points are tallied across the entire module. The final test can consist of any task elements, but most commonly it consists of, for example, consecutive Checkbox or Multichoice elements.</p><p>If the element is used in conjunction with a final test and only the final test points matter for passing the course, it is advisable to remove the points from task elements that are not part of the final test (set them to 0 points), so that only the test points are counted toward passing the course. You can change the points by selecting <a href="/help/69-moduulin-pisteiden-muokkaaminen-edit-points"><strong>Edit Points</strong></a> from the <strong>gear icon</strong> at the top of the module's edit view. You can also change the percentage pass threshold from the same view.</p><p>In a submodule structure with a test at the end, the elements included in the test and the <strong>Pass/Fail</strong> element are generally added to the main module after the submodule menu (<strong>Submenu Buttons</strong>).</p><p><strong>Note!</strong> The <strong>Pass/Fail</strong> element must <strong>not be added as the last element before the Finish element</strong>, as the Pass/Fail element will not function correctly in that case.</p><p>With additional configuration work, the <strong>Pass/Fail</strong> element can also be made to work at the end of submodules, so that the submodules must be passed successfully in order to proceed in the course. If there is no test at the end of the submodule structure and it is important that all submodule points are taken into account for passing, the <strong>Pass/Fail</strong> element is added as the last element in each submodule. In this case, the learner is directed from the submodule to the submodule menu (<strong>Submenu Buttons</strong>) only after passing the submodule. If the learner does not earn enough points from the submodule to pass it, they are directed back to the beginning of the submodule. <a href="/faq"><strong>Please contact Apprix</strong></a> if this functionality becomes necessary in your organization or if you would like to hear more about it.</p><p><strong>Note!</strong> The <strong>Pass/Fail</strong> element view appears to the learner <strong>only</strong> if the learner <strong>does not pass</strong> the final test.</p><p>When the learner has used up their possible test retake attempts, they must go back to review the course materials and start over from the beginning. The learner's final result as well as failed attempts are saved to the statistics. By using the <a href="/help/729-login"><strong>Login element</strong></a> and <a href="/help/82-user-authentication-kayttajatodennus"><strong>SSO feature</strong></a>, failed attempts can be filtered out when you want to view only the final result.</p><p><strong>Note!</strong></p><p>In <strong>Preview mode</strong>, <strong>module statistics are not saved</strong>. For this reason, the "<strong>Pass/Fail</strong>" and "<strong>Failure</strong>" elements do not work in preview mode either. <strong>Points are only calculated in Published mode.</strong></p><p>Why does Builder work this way? Because learners' <strong>completion data</strong> — not module <strong>testing data</strong> — is saved to the statistics.</p><h3>Element editing instructions</h3><ol start="1"><li><p>Open the element in edit view.</p></li><li><p>You can start by adding a background image to the element.</p></li><li><p>Then click the <strong>gear icon</strong> in the left corner of the element's edit view, which opens the element settings in a popup. In the settings you can change the following:<ul><li><p>the element/page from which the learner continues after a failed attempt (e.g. the first element of the final test)</p></li><li><p>the number of retake attempts</p></li></ul></p></li><li><p>Click on the element in the menu to which you want the learner to be directed to try again. The arrow should now move to that element.</p></li><li><p>Then select the number of retake attempts from the dropdown menu with an orange background.<ul><li><p>If the number of retake attempts is set to zero, the learner must restart the module from the beginning immediately after the first attempt.</p></li></ul></p></li><li><p>Click Done when the settings are complete.</p></li></ol><p>You can also edit the texts that appear after retake attempts. There are three text areas:</p><ol start="1"><li><p>The first area is visible on every failed attempt.</p></li><li><p>The second text area appears if the learner's next attempt also fails.</p></li><li><p>The third area appears when the learner must complete the course again from the very beginning.</p></li></ol><p>Note! The "Your points" field cannot be edited, as the learner's score is updated automatically in the field according to the score calculation.</p><p><strong>Also note that...</strong></p><ul><li><p>if the module contains <strong>Hotspot</strong> elements to which one or more <strong>True/False</strong> questions have been added, these points are available in the <strong>Edit Points</strong> window, but <strong>they are not calculated or considered in the Pass/Fail element</strong>.<ul><li><p>This means that in the Edit Points window, 0 points must be set for the Hotspot element, because the Edit Points feature counts the points from the Hotspot element's True/False questions, but the Pass/Fail element does not use or consider them when calculating the final score.</p></li></ul></p></li><li><p>if you add or remove items from a module or submodule that contains a Pass/Fail element, remember to re-check in the Pass/Fail element settings (gear icon in the upper left corner of the element view) the element/module point from which a new attempt should start.</p></li><li><p>when creating a module that contains a Pass/Fail or Pass/Fail Always element and you <strong>duplicate the module</strong>, <strong>check the restart point of the Pass/Fail element.</strong> In the duplicated module, the restart point for Pass/Fail and Pass/Fail Always elements cannot be found and defaults to the element immediately following the intro.</p></li></ul><h3>Pass/Fail Always</h3><p><strong>How does the Pass/Fail Always element differ from the Pass/Fail element?</strong></p><p>The <strong>Pass/Fail Always</strong> element is also shown after a successful test, when the course learner has earned enough points to pass the test. In the element, the learner also sees the number of correct answers/points relative to the maximum score.</p><h3>Element editing instructions</h3><ul><li><p>Click the gear icon in the upper left corner of the element view to change the settings, as in the <strong>Pass/Fail</strong> element (see the Pass/Fail element instructions earlier in this section).</p></li><li><p>Edit the text that is shown when the course learner has not passed the test, and the text that is shown when the course learner has passed the test.</p><p></p></li></ul><h3>Pass/Fail Always element: Learner did not pass the test / Learner passed the test</h3><p><img src="statamic://asset::files::docs/2021/09/PassFail_1_FI.jpg"></p><p><img src="statamic://asset::files::docs/2021/09/PassFail_Edit_FI.gif"></p><p><img src="statamic://asset::files::docs/2020/09/test-1.jpg"></p><p><img src="statamic://asset::files::docs/2020/09/passfail3-1.png"></p><p><img src="statamic://asset::files::docs/2021/10/PassFail_Always-1.png"></p><p></p>
