---
<p><b>Audit Module</b> – also known as the audit trail – is a chronological change history that records all changes made to a module in real time.</p><ul>
 <li><b>To activate the Audit Module feature:</b> click the module's <b>gear icon</b> in the edit view and select <b>Audit Module</b> from the dropdown. If the Audit Module feature is enabled, an <b>About Module</b> popup containing the change history will appear.</li>
 <li><b>To turn off the Audit Module feature:</b> click <b>Module Audit off</b> from the same dropdown. If the Audit Module feature is turned off, a popup notifying you of the deactivation will appear.</li>
 <li>To view the change history, select the <b>About this module</b> option from the gear icon dropdown in the module edit view. Alternatively, you can also view the change history by clicking the <b>Last modified</b> text link in the upper right corner of the module in the Menu view.
<ul>
 <li>If the <b>Audit Module</b> feature is enabled, the <b>About Module</b> popup containing the module's change history will open.</li>
 <li>If the <b>Audit Module</b> feature is not enabled, you will see a popup notifying you that the feature is not enabled.</li>
</ul>
</li>
 <li>Please note that the collected data can be downloaded in PDF, CSV, or Excel format.</li>
</ul><p>The example image on the page shows how changes made to module texts and images appear in the change history.</p><h3>User's Permissions Menu</h3><p>In the user management settings' <b>User's Permissions</b> menu, there are three permission options for using the <b>Audit Module</b> feature:</p><ul>

Row 7: The Finish element has moved position in the module from position 2 to position 4.</p><h3>Change history data recorded in the module</h3><p>The following information is recorded in the change history when the <b>Audit Module</b> feature is enabled:</p><ul></p>
 <li>Changed content in elements (e.g. text, images, videos)</li>
 <li>Added, deleted, or repositioned elements</li>
 <li>Name changes to modules and elements</li>
 <li>Enabling or disabling user passwords</li>
 <li>Published and unpublished modules (Published/Unpublished Modules)</li>
 <li>Duplicated modules (Module Duplicates)</li>
</ul><p><strong>Note</strong>! Remember to reload the Builder page so that the Administrate -&gt; Client -&gt; Features -&gt; Audit / Full on Publishing (to display archived files also incremental to be on) setting takes effect.</p><h3>Click images to enlarge them.</h3>
