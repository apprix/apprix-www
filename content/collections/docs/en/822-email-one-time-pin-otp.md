title: '8.2.2 Email One-Time PIN (OTP)'
---
<div class="fusion-text fusion-text-180">

This login (authentication) method eliminates the need for course participants to remember passwords. Instead, they simply enter their basic information (first name, last name, and email address) in the <strong>Welcome window</strong> each time they start the course.

<!--StartFragment -->

<strong>Activating the "Email One-Time PIN (OTP)" Authentication method</strong>
<ol>
 <li>Open the module in <strong>Edit mode</strong>.</li>
 <li>From the <strong>settings (gear) icon</strong> at the top of the module, choose <strong>User Authentication…</strong>. The <em>User Authentication &amp; Access</em> dropdown menu will open.</li>
 <li>Select <strong>Email One-Time PIN (OTP)</strong>. The chosen option will now appear in the gear icon dropdown menu.</li>
</ol>
<!--EndFragment -->

<strong>How it works</strong>
<ol>
 <li>The course participants fills the basic data (email, first name, last name) in the <strong>Welcome</strong> window and then clicks the <strong>Send PIN code</strong> button. A <strong>PIN code</strong> is automatically generated by the Authentication platform.</li>
 <li>The PIN code is sent to the participant via <strong>email</strong>.</li>
 <li>The participant pastes the PIN code into the <strong>authentication pop‑up window</strong>.</li>
 <li>The course is ready to start.</li>
</ol>

<strong>Important notes</strong>
<ul>
 <li>A new, unique <strong>PIN</strong> code is sent with each login attempt.</li>
 <li>The code must be entered within <strong>10 minutes</strong> of receipt.</li>
 <li>The user does not need to complete traditional registration.</li>
 <li>This method preserves the login across other <strong>OTP</strong>-enabled modules during the same session.</li>
 <li>It is especially recommended for external course participants and operational staff.</li>
</ul>
