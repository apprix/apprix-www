title: '8.2 User Authentication'

<p>In this section you will find the user authentication methods already implemented and integrated into Builder for identifying course participants.</p><p>Ask your <a href="/en/faq"><strong>Apprix contact person/account manager</strong></a> about possible implementation in your own Builder customer environment.</p><p>In the <strong>User Authentication</strong> settings, you can choose which user authentication method is used to identify the participant when they enter the module.</p><p>Below is a list of possible authentication methods:</p><p><strong>No user identification</strong></p><ul><li><p>Allows participants to take the course without logging in using any Builder login method.</p></li><li><p>Anyone who has received the module link can complete the training.</p></li><li><p>This method is most often used when the module contains a Login element or when the training can be completed anonymously and does not contain sensitive information.</p></li></ul><p><strong>Email registration (formerly "Consumer Login")</strong></p><ul><li><p>Allows participants to register in Builder and create a username and password.</p></li><li><p>The same username and password can be used in all modules (when email registration is enabled) after registration.</p></li><li><p>Requires the use of a phone or a valid personal email address.</p></li><li><p>Most often used for external participants and employees.</p></li></ul><p><strong>MyCourses LMS Sign-On</strong></p><ul><li><p>Allows access to the training from Apprix MyCourses LMS.</p></li><li><p>Requires the participant to first log in to MyCourses using the permitted login methods defined for the organisation.</p></li><li><p>Used when the module is placed in a MyCourses course as a language module.</p></li></ul><p><strong>Sympa ID check</strong></p><ul><li><p>The participant must enter a Sympa ID, local ID, first name, and last name.</p></li><li><p>The IDs must be found in the system and the names must match the information associated with the IDs.</p></li></ul><p><strong>Flexim tag reader</strong></p><ul><li><p>Participants can log in using a Flexim tag at a tag reader.</p></li><li><p>If a Flexim tag reader is not available, participants can log in using their employee ID, first name, and last name.</p></li><li><p>If using an employee ID, it must be found in the system and the name must match the information associated with the ID.</p></li></ul><p><strong>Single Sign-On (SSO)</strong></p><ul><li><p>Single sign-on <strong>(SSO)</strong> is a feature that allows logging in to Builder from many external services through a single login. The purpose is to avoid multiple successive logins and to streamline end users' use of application services.</p></li><li><p>Allows participants to log in automatically using company credentials.</p></li><li><p>Requires the participant to have credentials in an internal system (e.g. Active Directory, Google Workspace).</p></li><li><p>Most often used for internal employees.</p></li></ul><p><strong>Tax number check (vero.fi)</strong></p><ul><li><p>The participant must enter a tax number, first name, last name, and date of birth.</p></li><li><p>The tax number must be found in Finland's public tax number register. Please note that not all Finnish tax numbers are in the register – the person or their employer must add them to the public register.</p></li><li><p>The first name, last name, and date of birth must match the information associated with the tax number. If there are multiple first names, at least one must be entered. Three spelling mistakes are allowed in names.</p></li><li><p>Used when it is required that the company's employees' tax numbers are in the public register.</p></li></ul><p><strong>IP Guard</strong></p><ul><li><p>Allows access only if the participant comes from a specific IP address.</p></li><li><p>Used when the organisation uses a VPN. Access is only allowed if the participant has first logged in to the VPN network.</p></li><li><p>In some cases, also used to restrict access to a specific location, e.g. a factory or office building.</p></li><li><p>In some cases, also used to determine whether the participant is directed to single sign-on or not.</p></li></ul><p></p><p><img src="statamic://asset::files::docs/2024/04/User-Authentication.png"></p><p><img src="statamic://asset::files::docs/2024/04/User-authentication-settings.jpg"></p><p><img src="statamic://asset::files::docs/2022/03/Select_Domains-600x334.png"></p>
