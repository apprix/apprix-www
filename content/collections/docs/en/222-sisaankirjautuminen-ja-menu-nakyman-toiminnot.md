<p>If you are one of your organisation's administrators, you may also see the <strong>Administrate sidebar menu</strong> on the left side of the view, from which you can edit user management settings. Through the menu you can:</p><ul><li><p><p>create new users and edit users' login credentials</p><ul><li><p>change the username, email address, or password</p></li></ul></p></li><li><p><p>edit users' access permissions</p><ul><li><p><strong>Tip:</strong> If you are unsure which modules to grant a user access to, only grant access to the modules you are certain about.</p></li></ul></p></li><li><p>remove a user's access rights (deactivate a user)</p></li></ul><h3>How do I create a new user?</h3><ol start="1"><li><p>In the <strong>Administrate</strong> view, click the <strong>Add User</strong> button.</p></li><li><p>Fill in the required information and click <strong>Save.</strong></p></li><li><p><p>A popup window will appear informing you that the login credentials have been automatically sent to the new user's email address.</p><ul><li><p>The new user will receive two emails: one containing the username and Builder login link, and one containing the password.</p></li></ul></p></li><li><p><p>When the new user clicks the link in the email, they will be prompted to log in to Builder with their credentials and change their password. The user can change the password by selecting <strong>Change password</strong> from the account settings at the top of the <strong>Menu</strong> view.</p><ul><li><p>The new password should contain at least 8 characters, uppercase and lowercase letters, at least one number, and a special character.</p></li></ul></p></li><li><p>Once the new password has been set and the user clicks <strong>Continue</strong>, they will be notified that the password has been changed and the <strong>Menu view</strong> will open.</p></li></ol><p>Existing users can also change their password from the account settings in the Menu view by clicking <strong>Change Password</strong>.</p><h3>How do I change a user's access permissions?</h3><p>In the <strong>Administrate</strong> menu you can also change access permissions. Click the name of the user whose permissions you want to edit from the list. You will now see their user details, below which the module access permissions are displayed.</p><h3>User's Permissions menu</h3><ul><li><p><p><strong>User management:</strong></p><ul><li><p><strong>Edit option:</strong> the user has the right to add new users.</p></li><li><p><strong>None option:</strong> the user does not have the right to add new users.</p></li></ul></p></li></ul><ul><li><p><p><strong>Modules accessible:</strong></p><ul><li><p><strong>On option:</strong> the user can see modules in the All Modules window.</p></li><li><p><strong>Off option:</strong> the All Modules window is empty and the user cannot see any modules.</p></li></ul></p></li></ul><ul><li><p><p><strong>Default Module Access:</strong></p><ul><li><p><strong>None option:</strong> the user can only see their own modules.</p></li><li><p><strong>View (System Default) option:</strong> the user has view access to all existing and future modules, but no edit permissions.</p></li><li><p><strong>Edit option:</strong> the user has edit permissions for all existing and future modules.</p></li></ul></p></li></ul><h3>Administrator rights</h3><p>Grant administrator rights by selecting the <strong>Edit</strong> option from the <strong>User Management</strong> menu.</p><h3>General view access to modules</h3><p>Grant view access to all existing and future modules by selecting <strong>View</strong> from the <strong>Default Module Access</strong> menu.</p><h3>General module edit permissions</h3><p>Grant edit permissions for all existing and future modules by selecting <strong>Edit</strong> from the <strong>Default Module Access</strong> menu.</p><h3>Edit permissions for individual modules</h3><p><strong>(User's Module Edit &amp; Stats Permissions menu)</strong></p><p>Alternatively, as an administrator you can also grant a user access to only specific modules. Note, however, that all Builder users automatically have edit permissions for the modules they have created themselves. For other modules, an administrator can edit access permissions on a per-module basis by clicking <strong>Settings</strong> and selecting one of the following access permissions from the <strong>Access menu</strong> for the module in question:</p><ul><li><p><strong>None (System Default) option:</strong> The user cannot view or edit the module.</p></li><li><p><strong>View option:</strong> The user can view the module but cannot edit it.</p></li><li><p><strong>Edit option:</strong> The user can edit the module.</p></li></ul><p><strong>Statistics checkbox:</strong> You can grant a user permission to view the statistics of a specific module without granting edit permissions for that module. In this case, tick the checkbox and select <strong>View permissions</strong> from the module's <strong>Access menu.</strong> If you do not want to grant access to view statistics, leave the checkbox empty.</p><h3>Reset password</h3><p>Reset a user's password by clicking the <strong>Reset Password</strong> button. The user will receive two emails: one containing the username and Builder login link, and one containing the new password. Users can also change their own password at any time by selecting <strong>Change Password</strong> from the account settings in the <strong>Menu</strong> view at the top of the view.</p><h3>Removing access rights (deactivating a user)</h3><p>Deactivate and remove a user's rights by clicking the user's name in the Administrate menu whose access permissions you want to edit. Click the <strong>Deactivate User</strong> button. You can identify a deactivated user by the <strong>X</strong> symbol next to their name. You can reactivate the user at any time by clicking the user's name again and selecting <strong>Activate user.</strong></p><h3>How do you create a new user?</h3><p><img src="statamic://asset::files::docs/2025/03/2.2.2.-add-users-1.png"></p><h3><strong>How do you grant access permissions to a new user?</strong></h3><p><img src="statamic://asset::files::docs/2025/03/2.2.2.user-permission2-1.png"></p><h3><strong>Edit permissions for individual modules (User's Module Edit &amp; Stats Permissions menu)</strong></h3><p><img src="statamic://asset::files::docs/2025/03/2.2.2.-module-editing-permission-1.png"></p><h3><strong>Reset a user's password by clicking the Reset Password button.</strong></h3><p><img src="statamic://asset::files::docs/2025/03/2.2.2.-reset-password2-1.png"></p>
