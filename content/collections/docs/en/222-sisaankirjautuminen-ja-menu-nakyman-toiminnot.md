<p>If you are one of your organisation's admins, you may find <strong>Administrate </strong>from the sidebar on the left hand side of <strong>Menu</strong> view where you can edit user rights within your organisation. From the sidebar button you can:</p>

<ul>
 <li>create new users and edit user credentials
<ul>
 <li>change usernames, email addresses or passwords</li>
</ul>
</li>
 <li>edit user rights
<ul>
 <li><strong>Tip:</strong> If you are not certain which modules you should grant someone rights to, only grant rights to modules you are sure of.</li>
</ul>
</li>
</ul>

<h4 class="fusion-responsive-typography-calculated" data-fontsize="18" data-lineheight="27px"><strong data-fusion-font="true">How Can I Add a New User?</strong></h4>
<ol>
 <li>You can create new user accounts and define roles and permissions from the <strong>Administrate</strong> button in the <strong>Menu</strong> view. (Please note that the Administrate window is visible only to Builder’s Administrators.)</li>
 <li>From the <b>Administrate</b> window, click the <strong>Add User</strong> button and fill in the requested information (first name, last name, user ID, and email address). Then press <b>Save</b>.</li>
 <li>A pop-up window is displayed. It informs that the login information was sent via email to the new user, who will automatically receive two emails: one containing the user ID and Builder link and the second containing the Builder’s password.</li>
 <li>Once the new user clicks the link in the email, they are asked to log in to Builder and change the password. The user can change the password by selecting the option ‘<strong>Change Password</strong>’ from<b> </b>account settings at the top of the <b>Menu</b> view.
<ul>
 <li><strong>Note!</strong> The new password should contain at least eight characters, including upper- and lowercase letters, numbers, and special characters.</li>
</ul>
</li>
 <li>Once the new password is entered and the user clicks on the <b>Continue</b> button, they will be informed that the password was changed, and the <b>Menu </b>view will be opened.</li>
</ol>
<strong>Note!</strong> Older Builder users can change their own password by selecting ‘<strong>Change password</strong>’ from the <strong>Menu</strong> view.

<h4 class="fusion-responsive-typography-calculated" data-fontsize="18" data-lineheight="27px"><strong data-fusion-font="true">How Can I Assign the User Rights?</strong></h4>
The <b>Administrate</b> menu allows you to assign users' permissions. In the <strong>Users</strong> list, click on the user whose permissions you want to edit. You will now see his or her user name, under which you will see their user rights.

<ul>
 <li><b><strong>User management:</strong></b>
<ul>
 <li><b><strong>Edit option:</strong></b> the user has the right to add new users.</li>
 <li><b><strong>None option:</strong></b> the user is not allowed to add new users.</li>
</ul>
</li>
</ul>
<ul>
 <li><b><strong>Default Module Access:</strong></b>
<ul>

<ul>
 <li><b><strong>None option:</strong></b> the user can only see their modules.</li>
 <li><b><strong>View (System Default) option: </strong></b>the user has viewing rights to all existing and future modules but no editing rights.</li>
 <li><b><strong>Edit option: </strong></b>the user has editing rights to all existing and future modules.</li>
</ul>
</li>
</ul>
</li>
</ul>

<h4>Administrative Rights</h4>
Assign the administrative rights by selecting the <b>Edit</b> option from the <b>User Management</b> drop-down menu. Available options: None / Edit
<h4>Module Viewing Rights</h4>
Assign the viewing rights for all existing and future modules to a user by selecting <b>View</b> on the <b>Default Module Access</b> drop-down menu. Available options: View / Edit
<h4>Module Editing Rights</h4>
Assign the editing rights for all existing and future modules by selecting the <b>Edit</b> option from the <b>Default Module Access</b> drop-down menu. Available options: <em>None (System Default) /View/ Edit</em>.
<h4>Separate Module Viewing / Editing Rights</h4>
Alternatively, you can define which modules the user can view and edit. 

All regular users can edit only their modules. However, to give a user editing rights to some specific modules, select them from the <strong>User’s Module Edit &amp; Stats Permissions “Settings…</strong>” button. The <strong>User’s Module Edit &amp; Stats Permissions</strong> window opens.

Select the <b>Edit</b> option from the module’s <b>Access</b> drop-down menu. Now, the user can edit the module—available options: None / View / Edit. At the same time, you can also give the right to view the statistics. If you do not want this, please deselect the <b>Statistics</b> option. 

Builder will automatically save the settings.
<h4 class="fusion-responsive-typography-calculated" data-fontsize="18" data-lineheight="27px">Deactivate / Activate a User</h4>
<b></b>You can deactivate the user by clicking the username and then selecting the <strong>Deactivate User</strong> button. You can reactivate the user at any time by clicking again on the username and then selecting <strong>the Activate User</strong> button. You recognize that a user is deactivated since there is an “<b>X</b>” before the username. 
<h4 class="fusion-responsive-typography-calculated" data-fontsize="18" data-lineheight="27px">Reset Password</h4>
Click the Reset Password button to <strong>reset the user's password</strong>. Two emails will then be sent to the user. The emails contain the user's new password, user ID, and Builder link. The user can then choose the Change Password option to change <strong>her/his password</strong> at any time.
